我花了3小时重构个人网站，发现之前的架构全是坑

2026年1月22日 · 阅读需 6 分钟
阿男
Independent Developer @ RealTech Hub

我有个网站。 一个技术文档站。 放了一堆笔记。 看起来挺专业。

但我自己知道，这玩意儿就是个垃圾场。

一、

事情是这样的。

我之前用 Docusaurus 搞了个文档站。 想法很简单。 把平时写的技术笔记整理一下。 看起来像个正经的技术博客。

结果呢。

用了半年，我发现这东西根本不是我想要的。

首先，定位就不对。 它既不是产品文档，也不是系统教程。 就是一堆零散的笔记。 用文档的形式组织，看起来很别扭。

其次，我做了好几个产品。 比如 Peer Drop，一个 P2P 文件传输工具。 但网站上完全看不出来我在做什么。 就像一个只会写笔记的人。

最后，我发现我根本不想维护这个文档结构。 每次加内容都要想放在哪个目录。 太麻烦了。

我需要的是一个简单的博客。 加一个产品展示页面。 就这么简单。

但我一直没动手。 因为重构这种事，想想就头疼。

二、

今天下午突然想通了。

反正也没人看。 改坏了也无所谓。 干脆直接动手。

我打开终端。 跟 kiro-cli 说了一下想法。

我说：这个文档站我不想要了。 改成博客。 加个产品展示页面。 内容都保留。

它说：行。

然后就开始干了。

三、

第一步，把 docs 系统关了。

配置文件改了一行。 docs 插件直接禁用。

我当时还有点慌。 这么多文档，就这么没了？

但它说已经转成博客文章了。

我去看了一眼。 所有文档都变成了带日期的博客。 还自动加了标签。

Docker 系列，4篇。 DevOps 系列，3篇。 CI/CD 系列，2篇。

整整齐齐。

四、

然后开始搞产品展示。

它问我有什么产品。

我说就一个 Peer Drop。 P2P 文件传输工具。 WebRTC 做的。

它说够了。

然后就开始写。

创建了一个产品数据文件。 写了一个产品卡片组件。 做了一个展示页面。

那种带动画效果的卡片。 毛玻璃质感。 看起来还挺高级。

五、

改到一半，它问我要不要加个英文版。

我说那不是要翻译所有内容？

它说 Docusaurus 自带多语言。 配置一下就行。

然后就开始配置。

UI 文本提取出来。 生成翻译文件。 产品数据改成双语。

我试了一下语言切换。 居然真的能用。

英文版和中文版。 URL 自动分开。

六、

最后它问要不要换个设计。

我说行啊。 我一直挺喜欢 Supabase 那种深色风格。

它说那就用那个。

然后就开始改样式。

深色背景。 绿色主题。 渐变文字。 hover 动画。

刷新页面一看。 整个网站的气质都变了。

从朴素的文档站。 变成了现代化的个人站。

七、

整个过程用了多久？

3个小时。

我自己都没想到这么快。

最后看了一下 git 记录：

49个文件改了。 12517行新增。 6690行删除。

这要是我自己写。 光想想就累。

八、

改完之后，我突然意识到一个问题。

我之前为什么一直不想重构？

不是因为不会做。 是因为我知道这是个体力活。

要改配置。 要写组件。 要调样式。 要测试。 要处理各种边界情况。

这些事情，每一件都不难。 但加起来就很烦。

而今天能这么快搞定。 主要是因为我不用自己写代码。

我只需要说清楚想要什么。 剩下的事情，让 AI 去干。

九、

现在网站上线了。

有博客。 有产品展示。 有中英文切换。 有现代化的设计。

我终于有了一个像样的个人网站。

不是那种"看起来很专业但其实没什么用"的文档站。 而是一个真正能展示我在做什么的地方。

我把链接发给朋友。 他们说：哇，你什么时候变得这么专业了？

我说：就今天下午。

他们以为我在开玩笑。

但我是认真的。

十、

这件事让我想明白了一个道理。

很多时候，我们不做某件事。 不是因为不会做。 而是因为觉得麻烦。

比如重构代码。 比如写文档。 比如优化性能。

这些事情，我们都知道应该做。 但就是懒得做。

因为投入产出比不高。

但如果有工具能帮你处理这些体力活。 那这个投入产出比就变了。

原本需要1-2天的工作。 现在3小时就能搞定。

那些"想做但一直没做"的事情。 突然变得可行了。

十一、

当然，AI 也不是万能的。

它不能替你决定网站应该长什么样。 不能替你决定要做什么产品。 不能替你写出有价值的内容。

这些需要思考的事情，还是得你自己来。

但那些执行层面的事情。 那些重复性的工作。 那些"我知道怎么做但就是不想做"的事情。

可以让 AI 去干。

十二、

现在我的网站是这样的：

首页很简洁。 最新文章在上面。

What I Build 页面。 展示我做的产品。 每个产品都有卡片。 点进去就是产品链接。

Writing 页面。 所有技术文章。 按标签分类。 可以筛选。

右上角有语言切换。 中英文随便切。

整个设计是深色风格。 Supabase 那种感觉。 看起来还挺现代。

十三、

如果你也有个个人网站。 如果你也觉得它不够好。 如果你也一直想改但没动手。

可能不是因为技术问题。 而是因为你觉得麻烦。

试试让 AI 帮你。

不是说它能替你做所有事情。 而是说，它能处理那些你不想做的体力活。

你会发现。 很多改进，其实没那么难。

只是你一直没开始而已。
